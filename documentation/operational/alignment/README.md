## Scopo

Questa sezione fornisce una **fotografia del gap** tra:
- La specifica (`operational/spec/`)
- Lo stato corrente del repository

È il punto di partenza per:
- Identificare cosa manca
- Prioritizzare il lavoro
- Pianificare refactoring

## Uso Operativo

Prima di ogni task:
1. Leggi `01_executive_summary.md` per contesto
2. Consulta `03_gap_matrix.md` per problemi impattati
3. Verifica con `04_refactoring_priorities.md` se il tuo task è prioritario
